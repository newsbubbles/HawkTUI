## The Original Vision

From `notes/design/DESIGN.md`, HawkTUI was conceived as:

> A **premium terminal user interface** that wraps `pi_agent_rust`, transforming the already-powerful AI coding agent into an **immersive, visually stunning experience**. 3. **Keyboard-first**: Every action accessible without mouse

5. **Rust-native**: No subprocess wrapping - direct library integration

## What Makes HawkTUI Special?

### 1. **We Did It Right: Library Integration**

The most critical architectural decision was to integrate at the **library level**:

```rust
// src/providers/pi_bridge.rs
use pi::sdk::{
    AgentSessionHandle,
    SessionOptions,
    create_agent_session,
    all_tool_definitions,
};
```

Not a subprocess wrapper. Not an HTTP proxy. **Direct Rust library integration**.

This means:
- Sessions are shared with `pi` CLI
- Same configuration works
- Same tools available
- Zero IPC overhead
- Compile-time type safety

### 2. **Full Event Translation Layer**

```rust
fn translate_pi_event(event: &PiAgentEvent) -> Option<AgentEvent> {
    match event {
        PiAgentEvent::TurnStart { .. } => Some(AgentEvent::StreamStart { ... }),
        PiAgentEvent::MessageUpdate { assistant_message_event } => 
            translate_message_event(assistant_message_event),
        PiAgentEvent::ToolExecutionStart { .. } => Some(AgentEvent::ToolStart { ... }),
        // ... handles all pi events
    }
}
```

We built a complete event translation layer that maps pi's streaming events to HawkTUI's UI events.

### 3. **Theme System Done Properly**

```toml
# themes/hawk_dark.toml
[colors]
background = "#0d1117"
foreground = "#c9d1d9"
accent = "#58a6ff"

[colors.syntax]
keyword = "#ff7b72"
string = "#a5d6ff"
```

External TOML files, not hardcoded colors. Users can create custom themes.
